# Python Under the Hood

---

# 1. Python Is an Interpreted Language

Python is **not compiled directly into machine code** like C or C++.

Instead, Python code is executed in several steps.

### Execution Pipeline

```
Python Source Code (.py)
        ↓
Python Compiler
        ↓
Bytecode (.pyc)
        ↓
Python Virtual Machine (PVM)
        ↓
CPU Execution
```

Example Python code:

```python
x = 5 + 10
```

The Python interpreter compiles this into **bytecode instructions** which are executed by the **Python Virtual Machine (PVM)**.

---

# 2. CPython (The Main Python Implementation)

The most common Python interpreter is **CPython**.

CPython is written in the **C programming language**.

This means:

* Python syntax → compiled into bytecode
* Bytecode → executed by C code inside CPython

Example:

```python
print("Hello")
```

Internally calls a **C function inside CPython**.

Many Python libraries also rely heavily on C for performance.

---

# 3. Why Python Is Slower Than C

Python is slower because it is:

* dynamically typed
* interpreted
* memory managed

Example in Python:

```python
a = 10
b = 20
c = a + b
```

Python must determine at runtime:

* type of `a`
* type of `b`
* type of operation

In C:

```c
int a = 10;
int b = 20;
int c = a + b;
```

Types are known at compile time, so execution is faster.

---

# 4. Python Objects

In Python, **everything is an object**.

Examples:

| Value   | Object Type |
| ------- | ----------- |
| 5       | int         |
| 3.14    | float       |
| "hello" | str         |
| [1,2,3] | list        |
| {}      | dict        |

Even functions and classes are objects.

Example:

```python
def greet():
    pass
```

`greet` itself is a Python object.

---

# 5. What a Python Object Contains

Internally, every Python object contains:

```
Object
│
├── Reference count
├── Type information
└── Value
```

Example:

```
Object: "hello"

Type: string
Value: hello
Ref count: number of references
```

Reference counting helps Python manage memory automatically.

---

# 6. Strings Under the Hood

Python strings are **immutable objects**.

Example:

```python
text = "hello"
```

Memory representation:

```
string object
│
├── length
├── encoding
└── characters
```

If you modify a string:

```python
text = text + " world"
```

Python creates a **new string object** instead of modifying the original.

---

# 7. Lists Under the Hood

Python lists are **dynamic arrays**.

Example:

```python
numbers = [1,2,3]
```

Internally:

```
list object
│
├── pointer to array
├── capacity
└── length
```

Lists grow dynamically when new elements are appended.

Example:

```python
numbers.append(4)
```

Python may allocate a larger memory block behind the scenes.

---

# 8. Dictionaries Under the Hood

Python dictionaries are implemented as **hash tables**.

Example:

```python
data = {"name": "Alice"}
```

Structure:

```
hash table
│
├── key hash
├── key
└── value
```

Dictionary lookup is typically **O(1)** on average.

This is why dictionaries are extremely fast for key lookups.

---

# 9. NumPy Under the Hood

NumPy is fast because it is **written mostly in C**.

Python code:

```python
import numpy as np
a = np.array([1,2,3])
```

Internally:

```
Python API
   ↓
NumPy C implementation
   ↓
Optimized CPU operations
```

This avoids Python loops and uses **vectorized operations**.

---

# 10. Pandas Under the Hood

Pandas is built on top of:

* NumPy
* C extensions
* Cython

Example:

```python
df["age"].mean()
```

Internally this calls **optimized C code**, not pure Python.

---

# 11. GIL (Global Interpreter Lock)

Python has a **Global Interpreter Lock (GIL)**.

The GIL ensures:

```
Only one thread executes Python bytecode at a time.
```

This simplifies memory management but limits CPU parallelism.

That is why Python uses:

* multiprocessing for CPU tasks
* threading for I/O tasks

---

# 12. Why Libraries Like PyTorch Are Fast

Libraries such as:

* PyTorch
* TensorFlow
* NumPy

are fast because they use:

* C
* C++
* CUDA

Example:

```
Python API
   ↓
C++ backend
   ↓
GPU kernels
```

Python acts as a **high-level interface**.

---

# 13. Python Memory Management

Python manages memory automatically using:

1. Reference counting
2. Garbage collection

Example:

```python
a = [1,2,3]
b = a
```

Both variables reference the same object.

When references drop to zero, the object is removed.

---

# 14. Why Python Is Still Popular

Even though Python is slower than compiled languages, it is popular because:

* simple syntax
* huge ecosystem
* rapid development
* powerful libraries

In most ML workflows:

```
Python orchestrates
C/C++ libraries execute heavy computations
```

---

# Summary

Understanding Python internals helps developers write better code.

Key concepts:

| Concept        | Meaning                           |
| -------------- | --------------------------------- |
| CPython        | Python interpreter written in C   |
| Bytecode       | intermediate instruction set      |
| Python objects | everything in Python is an object |
| Lists          | dynamic arrays                    |
| Dictionaries   | hash tables                       |
| NumPy          | C-based numerical engine          |
| GIL            | limits multithreading execution   |

Python acts as a **high-level control language**, while performance-critical work is handled by optimized native libraries.
